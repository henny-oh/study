Microsoft Azure Virtual Training Day
온-프레미스 인프라 및 데이터 마이그레이션

1.클라우드 마이그레이션 계획 평가
-마이그레이션 요구사항
web apps, sql 인스턴스 azure로 마이그레이션

-마이그레이션 대상
    -클라우드 전용
    1)모든 워크로드를 클라우드로
    2)온프레미스 azure directory domain services에서 마이그레이션
    3)최종 대상 : azure에서 실행되는 모든 서버 워크로드가 있는 온 프레미스로 제공되는 클라이언트 운영 체제만
    -하이브리드
    1)일부 워크로드를 azure로 마이그레이션
    2)하이브리드 id
    3)윈도우 서버에서 호스팅되는 일부 온-프레미스 워크로드를 azure stack과 같은 관리형 시스템으로 마이그레이션
    4)azure 또는 관리형 시스템으로 마이그레이션할 수 있을 때까지기존 온프레미스 워크로드의 일부 유지

-caf(클라우드 채택 프레임워크)
    -가속화 전략
    -azure 랜딩 존

-azure migrate
온프레미스 서버, 인프라, 어플리케ㅣ션 및 데이터를 평가하고 azure로 마이그레이션하는 중앙 집중식
    -통합 도구
    서버평가, 서버 마이그레이션, 데이터베이스병가, auzre로 마이그레이션, 서버 평가, 웹앱 평가 및 azure로 마이그레이션, 오프라인 데이터 마이그레이션
-클라우드 마이그레이션 오해
    1)클라우드는 항상 저렴
    2)모든 것을 클라우드로 마이그레이션해야한다
    3)온프레미스 환경을 미러리하면 클라우드에서 비용을 절감할 수 있다.
    4)서버 비용이 클라우드 마이그레이션에 대한 비즈니스 사례를 주도한다.
    5)운영 비용 모델이 자본 비뵹 모델 보다 낫다
    6)클라우드 마이그레이션은 스위치를 누르는 것 만큼 간단하다 
-ammp(azure 마이그레이션 및 현대화 프로그램) : 교육 및 전문가 도움으로 자신있게 마이그레이션 지원
    
    =>플래닝이 중요함
2.window server 컴퓨팅 마이그레이션
-온프레미스 데이터센터를 azure로 마이그레이션
    -컴퓨팅 인스턴스 검색 : vm, hyper,a물리적 서버, aws,gcp 인스턴스 검색(서버 평가 도구 사용)
    -평가 :azure 준비상태, 크기, 비용, 종속성 분석
